E-voting In Blockchain

The aspect of security and transparency is a threat from still widespread elections with the conventional voting system. General elections still use a centralized system, there is one organization that manages it which has full control over the voting process.



What is Blockchain

A blockchain is, in the simplest of terms, a time-stamped series of immutable records of data that is managed by a cluster of computers not owned by any single entity. Each of these blocks of data (i.e. block) is secured and bound to each other using cryptographic principles (i.e. chain).
So, what is so special about it and why are we saying that it has industry-disrupting capabilities?
The blockchain network has no central authority — it is the very definition of a democratized system. Since it is a shared and immutable ledger, the information in it is open for anyone and everyone to see. Hence, anything that is built on the blockchain is by its very nature transparent and everyone involved is accountable for their actions.


Why Blockchain

An election is a major concern for a healthy democracy. Fair and transparent elections at low cost are the main aim of this system. With the use of blockchain and cryptography, end to end verifiable voting systems can be implemented. Blockchain technology has many advantages like security and transparency, while homomorphic encryption ensures the anonymity of voters.

Eligibility: Only the person who has registered himself/herself with the Election Commission of India is eligible to cast a vote for an election.

Vote Verifiability: Each voter can verify whether his vote is cast and counted correctly.

Result Verifiability: Anyone can verify whether the tallying of votes is done correctly.

Paper Free: The voting process does not include any use of paper for confirming the vote to the voter which eliminates any chances of vote-buying.

Coercion-Resistance: Coerced voting is not possible since the coercer cannot identify the candidate to which the voter has cast his vote, on the other hand, the voter can easily prove that he voted for the coercer’s candidate.

Integrity: It makes sure that the casting of the vote is accurate and the tallying process is consistent.

Voter Privacy: The privacy of voters is maintained since no one can trace the votes to corresponding voters.


Advantages of Blockchain

Blockchain technology is one of the solutions to overcome these problems in the voting system because it embraces a decentralized system and the entire databases are owned by many users. It enables a group of people to maintain a public database that is owned, updated, and maintained by every user, but controlled by no one. Blockchain-based implementation will remove the issue of database manipulation. So blockchain technology has the potential to make e-voting more acceptable and reliable.


Homomorphic Encryption

Homomorphic encryption is used to make sure that votes hold its conﬁdentiality by encrypting and calculating all votes without decrypting. Homomorphic encryption is an encryption method which enables any data to remain encrypted while it is being processed and manipulated. This lets you or a third party use encrypted data functions without exposing data values. It allows for the manipulation and study of encrypted data as if they were in plaintext format without decrypting them.

For example, if the dynamically generated Reference String (R) for 4 candidates standing in an election is (0,1,1,0). Where,
0 is the reference bit for the 1st candidate.
1 is the reference bit for the 2nd candidate.
1 is the reference bit for the 3rd candidate.
0 is the reference bit for the 4th candidate.
Let’s assume the randomly selected set by the system is “A” and the voter “V” wants to vote for the 3rd candidate ( i.e. “Yes” for the 3rd candidate and “No” for all the other candidates) whose reference bit is 1.
Now in order to generate the “β string”, set “A” and the Reference String (R) i.e. (0,1,1,0) will be used.
The “β” string is generated as follows using set “A” from the Reference Table mentioned above:
For 1st candidate, the reference bit is 0 and the candidate does not receive the vote from voter "V", i.e. a "NO" so the beta bit generated is 0.
For 2nd candidate, the reference bit is 1 and the candidate does not receive the vote from voter "V", i.e. a "NO" so the beta bit generated is 1.
For 3rd candidate, the reference bit is 1 and the candidate receives the vote from voter "V", i.e. a "YES" so the beta bit generated is 0.
For 4th candidate, the reference bit is 0 and the candidate does not receive the vote from voter "V", i.e. a "NO" so the beta bit generated is 0.
Hence, after combining all the generated beta bits i.e. 0, 1, 0 and 0, the final “β string” becomes (0,1,0,0).
